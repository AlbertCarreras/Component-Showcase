# Design Principles

Guidelines for using and extending this design system.

## Consistency

Use the provided components and tokens instead of one-off styles. This keeps the product visually consistent and makes future theme changes easier.

## Accessibility

- Use semantic HTML (buttons for actions, links for navigation).
- Ensure sufficient color contrast (tokens are chosen with this in mind).
- Support keyboard and screen reader usage.

## Simplicity

Components expose a small, clear API. Prefer composition over adding many props.
